# Functional Blocks Grouped By Potency

Potency means progression leverage / bypass risk. Higher tiers are more likely to unlock resource transformation, automation, energy, logistics, magic processing, networked crafting, or dimension/world access.

The grouping is heuristic because the runtime `registries.json` dump is empty. It is based on the previous functional block candidates, block IDs from blockstates, and recipe/process signals from `recipes.json`. Decorative shape variants are deliberately demoted unless they are explicit machines/controllers.
